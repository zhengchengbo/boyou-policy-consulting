## 一、产品定位与运营规范（introduction + product）

### 注册与审核
- 企业主体可选对公打款（免费）或微信认证（300元/年）
- 个人主体最多15个项目成员+15个体验成员
- 审核通过后需手动点击发布才上线
- 提交审核时需配置功能页面（不超过5组）和测试账号

### 运营红线（product规范）
- **禁止强制分享**：分享后才能解锁功能、弹窗诱导分享 → 限制分享能力
- **禁止诱导分享朋友圈**：分享获利益 → 封禁
- **禁止收集用户隐私**：不得要求用户输入手机号等才可用功能
- **禁止类目不符**：服务必须与选择的类目一致，否则限期整改
- **核心功能必须在首页体现**（规范3.2）
- **名称不得含商业化用语**："免费""巨便宜"等
- **UGC内容必须接入内容安全检测接口**（规范5.18）
- **禁止滥用模板消息/订阅消息**：诱导订阅会被限制接口
- **禁止混淆官方**：不得仿冒微信官方功能或政府机构
- **虚拟支付必须接入小程序虚拟支付**，不得引导至外部支付

### 用户隐私保护
- 采集数据前必须用户同意并如实披露用途
- 不得向其他用户显示微信号/手机号等信息
- 终止运营后应立即删除用户数据

## 二、设计规范（design guide）

### 核心设计原则
1. **友好礼貌**：重点突出、流程明确，减少干扰元素
2. **清晰明确**：导航明确、反馈及时、异常可控
3. **便捷优雅**：减少输入、避免误操作（热区7-9mm）、善用接口
4. **统一稳定**：不同页面间保持一致的控件和交互方式

### 视觉规范要点
- 常用字号：22, 17, 15, 14, 12(pt)
- 设计稿基准宽度：375px（固定布局）或 390px（响应式布局）
- 右上角小程序菜单占位需预留（iOS右滑返回、安卓物理键返回）
- 加载反馈：避免模态加载，推荐局部加载；不要超过1个加载动画
- 结果反馈：局部操作→局部反馈；全局操作→弹出式提示/模态框/结果页
- 异常状态：必须告知原因+解决方案，不能死页面
- 底部Tab标签2-5个，建议不超过4个
- 微信官方设计组件库 WeUI：https://weui.io

## 三、开发框架（framework）

### WXML
- 数据绑定：`{{message}}` 双花括号
- 列表渲染：`wx:for="{{array}}"`，默认 item/index
- 条件渲染：`wx:if / wx:elif / wx:else`
- 模板：`<template name="xxx">` + `<template is="xxx" data="{{...obj}}">`

### 页面注册
- 简单页面用 `Page({})` 构造器
- 复杂页面可用 `Component({})` 构造器（支持behaviors等高级特性）
- 生命周期：onLoad → onShow → onReady
- 数据更新：`this.setData()` 单向数据流
